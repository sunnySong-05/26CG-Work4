# 实验报告：基于 Taichi 的光线投射与 Phong 着色实现

## 一、项目介绍

本实验旨在利用 Taichi 编程语言，通过代码驱动的方式在三维空间中构建几何场景。实验不依赖外部 3D 模型文件，而是通过数学隐式方程定义几何体，并利用光线投射技术进行渲染。

实验的核心任务包括：
1. 隐式几何建模：在 Taichi Kernel 中利用光线投射数学隐式定义红色球体与紫色圆锥。
2. 深度测试：实现类似 Z-buffer 的竞争逻辑，确保射线击中多个物体时选择最近交点，保证正确遮挡。
3. 编写 Phong 着色器：根据交点坐标计算 L、V、R 向量，并合成 Ambient、Diffuse 和 Specular 颜色。
4. 完成 UI 交互面板：创建滑动条控件，实现着色器参数的实时动态更新。


## 二、项目架构

### 2.1 技术栈
* 语言: Python 3.x
* 计算框架: Taichi Graphics（高性能并行计算）
* GUI 库: Taichi 内置的 ti.ui 模块

### 2.2 目录结构
由于本项目为代码驱动的单文件实现，结构如下：
* main.py: 包含交互参数场定义、几何求交逻辑、渲染内核以及 UI 窗口主循环。


## 三、代码逻辑

本实验的代码逻辑遵循标准的光线投射管线，并在 Taichi 的并行框架下加速执行：

1. 场景初始化与场定义：
   使用 ti.field 定义像素缓冲区和交互参数（ka, kd, ks, shininess），确保存储在全局字段中以便 Kernel 访问。

2. 数学建模与求交：
   * 球体：解一元二次方程判断射线与球面的交点。
   * 圆锥体：利用圆锥侧面方程计算交点，并通过 y 轴高度范围进行截断。

3. 渲染核心：
   * 射线生成：根据像素位置计算归一化方向。
   * 深度竞争：比较射线与球体和圆锥的距离 t，选取最小的正数 t 值，并据此计算击中点的世界坐标和法向量 N。

4. Phong 着色计算：
   在最近交点处，计算光源方向 L、观察方向 V 以及反射方向 R。应用公式叠加环境光、漫反射和高光分量，最终输出 RGB 颜色。


## 四、实现功能

* 代码驱动场景：无需加载外部模型，完全由数学逻辑生成红色球体和紫色圆锥。
* 深度测试：成功实现深度竞争逻辑，物体间的遮挡关系准确。
* 完整 Phong 光照：支持实时调整环境光、漫反射、镜面高光强度及高光指数。
* 实时交互：通过 UI 面板的 Slider 控件，可即时观察材质属性变化对渲染效果的影响。


## 五、实验结果展示

* 场景布局：左侧放置红色球体，右侧放置紫色圆锥，背景采用深青色。
* 视觉效果：在点光源照射下，物体表面呈现出明显的明暗变化与高光点。
* 交互观察与光线变化说明：
    * 调节 Ka (环境光系数)：增加该值会提升物体背光面和阴影区域的整体亮度，使物体在没有直接光照的情况下也能显现基础轮廓。
    * 调节 Kd (漫反射系数)：该参数决定了物体表面对光线的散射程度。增大 Kd 会增强物体受光面的色彩浓度和体感，使球体和圆锥更具实体感。
    * 调节 Ks (镜面高光系数)：增大 Ks 后，物体表面受光处会出现明显的亮斑。该值越大，高光区域越亮，使材质看起来更趋向于光滑的金属或塑料。
    * 调节 Shininess (高光指数)：这是观察效果最明显的变化。当该值较低时，高光点呈现大面积的模糊状态；随着 Shininess 增大，高光点迅速聚拢并变小，表现出极度光滑表面的视觉特征。
* 性能表现：在 1200 x 1200 分辨率下，Taichi 提供了流畅的交互体验，参数更新无延迟。

## 结论:

通过本次实验，我掌握了在 Taichi 中利用数学隐式方程构建三维场景的方法，深入理解了光线投射算法与 Phong 着色模型。实验证明了代码驱动建模在处理规则几何体时的灵活性，同时也展示了并行计算在图形渲染任务中的巨大优势。
